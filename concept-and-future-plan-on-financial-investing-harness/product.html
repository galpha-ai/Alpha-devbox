Product

DevBox + AlphaDevBox

DevBox is the open-source platform: a Cloud Managed Agent that runs on Google Cloud and exposes a real tool layer to the model. AlphaDevBox is the financial-research surface built on top of it — the hosted AI researcher a discretionary trader actually uses.

Two layers of the same product

Platform

DevBox

Open-source Cloud Managed Agent. Built on Google Cloud's Cloud Agent SDK. Hosts a persistent agent, the user's own Cloud Agent SDK, and whichever front-end connections a team wants.

  • Persistent hosted runtime
  • Tool layer for applied math, statistics, and quantitative packages
  • Multi-user sessions wired into Slack / Telegram
  • Versioned research environment with shared facts
Vertical surface

AlphaDevBox

The financial-research layer. The demo a trader sees: a hosted AI researcher that takes natural language and runs the research end to end.

  • Backtest any idea in natural language
  • Simulate any scenario — including ones without a historical analog
  • Small-size live deployment with human-in-the-loop escalation
  • Shared research graph across a team

The three-layer stack

Layer A

Forecast & Backtest

For any problem with a clear target and enough history.

  • Turn a view into a forecast ŷₜ.
  • Run rolling regression, tree models, predictive DNNs — walk-forward by default.
  • Trigger-based strategies: price, volume, earnings, news, tweets, prior model outputs.
  • Standard outputs: Sharpe, turnover, drawdown, per-sector attribution, failure-mode table.
Layer B

Scenario & Simulation

For problems that don't have a clean historical analog — catalyst bets, regime shifts, asymmetric payoff construction.

  • Bear / base / bull case with explicit payoffs. Default uniform probability; user can override.
  • Kelly-style sizing on the mapped payoff × probability.
  • Multi-agent what-if simulations, reproducible from a seed.
  • Prediction-market bridge so probabilities are anchored in something external.
Layer C

Operator — human-in-the-loop

For actually deploying capital without turning the user into a full-time trader.

  • Shadow mode and small-capital live deployment to measure the sim-to-real gap.
  • Automation threshold (default $1,000). Below: automated. Above: escalate to human.
  • Per-strategy dashboards: sim vs live PnL, fills, spreads, market impact, adverse selection.
  • Process validation on every closed trade: did the stock move for the reason we predicted?
Research direction

Financial World Model

Traditional "world models" in quant are market-impact models — useful but local. What is missing is a model of the causal relations between events: macro prints, earnings, geopolitics, sector narratives. Year one builds the scaffolding; a full model is a multi-year program.

Why this is not a coding agent

A generic coding agent writes code. AlphaDevBox calls code. The research workflow — the data, the toolbox, the simulator, the operator — lives inside the agent harness, not outside it. The user's natural-language intent becomes a real research procedure, not a throwaway script.

What lives inside

Tool layer

  • Applied mathematics and statistics libraries
  • Quantitative research packages
  • Rolling regression and hypothesis testing toolboxes
  • ClickHouse historical data
  • Financial historical simulator
  • Persistent state across long research sessions
What we get from it

Research primitives

  • Natural-language backtests with point-in-time correctness
  • Scenario runs that can be replayed after the event
  • Asymmetric bet construction as a first-class operation
  • Process-based validation alongside realised PnL
  • Shared-facts collaboration across a multi-user team

Collaboration model

The mental model is a group of programmers doing compound engineering in a shared Slack — except the work product is research on historical financial data rather than code. Multiple users join the same hosted DevBox session. They share the same tool layer, the same data, the same facts. Every idea, backtest, scenario, and postmortem is a node on a research graph. New members inherit the state of the lab instead of starting from an empty spreadsheet.

Integration path

Preferred: our hosted product

DevBox on Google Cloud + persistent runtime + our tool layer + multi-user collaboration. This is where the full stack shows up, including the operator and the research graph.

Alternative: CLI into your own coding agent

The ClickHouse historical data and the financial historical simulator can also be exposed as a CLI that plugs into external coding agents. It works — but it trades away the shared environment, which is the piece we care about most.

Open-source core

DevBox itself is open source. The open-source piece is not any single feature — it's the framework: take the applied math, statistics, and quantitative-research tooling that a researcher would normally run by hand, and expose it as a reusable tool layer inside an agent harness running on a persistent cloud runtime. The hosted AlphaDevBox, the collaboration layer, the financial data, and paid access sit on top.
